# Sadie's Schedule — Setup Guide

## What's in this folder

| File | Purpose |
|---|---|
| index.html | The entire app — open this in a text editor to make config changes |
| manifest.json | PWA metadata — controls the home screen icon name |
| sw.js | Service worker — enables offline mode |
| netlify.toml | Netlify deployment config — do not edit |
| icon-192.png | App icon (you must add this — see Step 3) |
| icon-512.png | App icon (you must add this — see Step 3) |

---

## Step 1 — Google account

You need a Google account to use Firebase.
Create one at: https://accounts.google.com/signup
(Skip if you already have one.)

---

## Step 2 — Firebase setup (~3 min)

### 2.1 — Create the project
1. Go to: https://console.firebase.google.com
2. Sign in with your Google account
3. Click "Add project"
4. Name it: sadies-schedule
5. Disable Google Analytics when prompted (not needed)
6. Click "Create project" and wait for it to finish

### 2.2 — Create the Realtime Database
1. In the left sidebar, hover to expand it if collapsed
2. Click "Build" to expand that section
3. Click "Realtime Database"
4. Click "Create database"
5. Region: choose us-central1
6. Security rules: choose "Start in test mode"
7. Click "Enable"

### 2.3 — Set database rules
After the database is created:
1. Click the "Rules" tab at the top of the Realtime Database page
2. Replace whatever is there with:

{
  "rules": {
    ".read": true,
    ".write": true
  }
}

3. Click "Publish"

### 2.4 — Register a web app and get your config
This step generates the connection credentials that go into index.html.

1. Click the gear icon (⚙) in the top-left of the Firebase console,
   next to your project name, then click "Project settings"

2. Scroll down to the "Your apps" section.
   - If you see a row of icons (<> iOS Android Web), click the </> icon (Web)
   - If you already see an app listed there, click its name to expand it
     and skip to step 4

3. In the "Add Firebase to your web app" dialog:
   - App nickname: type sadie-cal
   - Leave "Also set up Firebase Hosting" UNCHECKED
   - Click "Register app"

4. Firebase shows you a code block that looks like this:

   const firebaseConfig = {
     apiKey: "AIzaSy...",
     authDomain: "sadies-schedule.firebaseapp.com",
     databaseURL: "https://sadies-schedule-default-rtdb.firebaseio.com",
     projectId: "sadies-schedule",
     storageBucket: "sadies-schedule.appspot.com",
     messagingSenderId: "123456789",
     appId: "1:123456789:web:abc123"
   };

   Copy this entire block. Every value between the { } is needed.
   The databaseURL line is especially important — it will not appear
   unless you created a Realtime Database first (Step 2.2).
   If databaseURL is missing, go back and complete Step 2.2, then
   return to Project Settings and it will appear.

5. Open index.html in a text editor (TextEdit on Mac works, or
   any code editor like VS Code)

6. Find the section that says:
   🔧 PASTE YOUR FIREBASE CONFIG BELOW

7. Replace the entire firebaseConfig block (the 7 PASTE_YOUR_... lines)
   with the one you copied from Firebase

8. Save the file

## Step 3 — App icons

You need two PNG image files in this same folder:
  icon-192.png  (192 x 192 pixels)
  icon-512.png  (512 x 512 pixels)

These appear as the home screen icon when someone installs the app.

Free option — generate from an emoji:
  Go to: https://favicon.io/emoji-generator
  Pick an emoji (📅 works well)
  Click Download
  Inside the zip you'll find files named favicon-192x192.png
  and favicon-512x512.png — rename them to icon-192.png and
  icon-512.png and place them in this folder.

---

## Step 4 — Deploy to Netlify (~2 min)

1. Create a free account at: https://netlify.com
2. After signing in, you land on the "Sites" dashboard
3. Drag this entire "Sadies Schedule App" folder onto the
   "Deploy manually" drop zone on that page
4. Netlify processes it and gives you a URL like:
     https://random-name-123.netlify.app
5. Optional: click "Site configuration" → "Change site name"
   to get something like sadies-schedule.netlify.app
6. Share that URL with Chris and Jess

---

## Step 5 — First use (PIN setup)

Each person opens the app URL the first time:
1. They pick their name from the dropdown (Chris, Trevor, or Jess)
2. Because no PIN exists yet for them, they see a "Create your PIN" screen
3. They tap any 4 digits on the number pad — that becomes their PIN
4. Their PIN is stored (hashed) in Firebase
5. From then on, choosing their name will show the PIN entry screen instead

If someone forgets their PIN, you can reset it by going to your
Firebase Console → Realtime Database → Data → expand "pins" →
delete the entry for that person's name. They'll be prompted to
create a new PIN next time they sign in.

---

## Step 6 — Install as an app on each phone

iPhone / iPad:
  Open the URL in Safari (must be Safari, not Chrome)
  Tap the Share button (box with arrow pointing up)
  Scroll down and tap "Add to Home Screen"
  Tap "Add" — the icon appears on the home screen

Android:
  Open the URL in Chrome
  Tap the three-dot menu (top right)
  Tap "Add to Home Screen" or "Install app"
  Tap "Add" or "Install"

Once installed it runs full-screen with no browser address bar,
just like a native app. It also works offline using cached data.

---

## Making changes later

Open index.html in any text editor. The "QUICK-EDIT CONFIG" comment
block near the bottom of the file (search for that phrase) documents
exactly where to change:

  - The app title and tagline
  - User names in the login dropdown
  - Custody party colors and labels
  - The child's name
  - The Firebase config (if you ever recreate the project)

After any change, re-deploy by dragging the folder to Netlify again.
Netlify replaces the old deploy automatically.
